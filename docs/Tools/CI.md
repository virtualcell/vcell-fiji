# Continuous Integration/ Continuous Development

When developing these applications for smooth development lifecycle GitHub actions
will be used for CI/CD. To help with this process for SciJava development there is both a [wiki](https://wiki-biop.epfl.ch/en/IPA/version-control)
and a [GitHub repository](https://github.com/scijava/scijava-scripts/tree/main) to aid in the process.



### CI
For continuous integration automation of tests, building, and branch merging occurs
within github actions.

### CD
When a new version of an application is ready to be deployed,
the process of testing, building, and deploying is all done
automatically.
